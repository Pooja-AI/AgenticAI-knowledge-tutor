# Executor

## Overview

An Executor is the component of an AI agent responsible for carrying out the execution plan created by the Planner. It performs the actual work by invoking tools, calling APIs, querying databases, retrieving information, executing code, and collecting results.

While the Planner decides **what should be done**, the Executor determines **how each task is performed**. It bridges the gap between reasoning and action.

In Agentic AI, the Executor enables autonomous task completion by interacting with external systems and returning results back to the LLM for further reasoning or response generation.

```python

    "Search flights",
    "Book hotel",
    "Generate itinerary"
```

# Executor in Agentic AI

In Agentic AI, the Executor transforms plans into actions. After receiving an execution plan from the Planner, it identifies the appropriate tools, APIs, databases, or code execution environments needed for each task. It performs the required operations, collects outputs, handles failures, and returns results to the LLM for further reasoning or to generate the final response. This separation of planning and execution enables intelligent, modular, and scalable AI agents.

---

# Executor vs Planner

| Executor | Planner |
|-----------|----------|
| Executes tasks | Creates execution plans |
| Invokes tools and APIs | Breaks goals into subtasks |
| Performs actions | Performs reasoning |
| Collects results | Determines execution order |
| Focuses on implementation | Focuses on strategy |

# Summary

The Executor is a core component of an AI agent responsible for carrying out the execution plan created by the Planner. By invoking tools, calling APIs, accessing databases, executing code, and collecting results, it enables AI agents to perform real-world actions autonomously. Working together with the Planner, Memory Module, Retriever, and Tool Manager, the Executor forms the operational engine of modern Agentic AI systems.
